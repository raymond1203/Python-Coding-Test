# Python 3.14 Baseline

> 이 레포의 Python Basics는 **Python 3.14.6**을 기준으로 작성합니다.  
> 단, 특정 프로젝트나 실행 환경이 다른 Python 버전을 사용할 수 있으므로 버전 의존 동작은 별도로 표시합니다.

## Current Reference

- Python 3.14.6 official documentation: <https://docs.python.org/3/>
- What's New in Python 3.14: <https://docs.python.org/3/whatsnew/3.14.html>
- Python documentation by version: <https://www.python.org/doc/versions/>

## Why This Matters

`Effective Python` 3rd Edition은 공식 소개 기준 Python 3.13까지를 다룹니다. 이 레포는 같은 공개 목차를 학습 체크리스트로 사용하되, 실제 설명과 예제는 Python 3.14.6 기준으로 재검증합니다.

중요한 점은 “최신 문법을 많이 쓰는 것”이 아닙니다. Python 3.14.6 기준으로 언어의 현재 모습을 이해하되, 언제 보수적인 문법을 선택해야 하는지도 판단하는 것입니다.

## Python 3.14에서 특히 확인할 항목

| Area | Why it matters |
|---|---|
| Deferred annotations | 타입 힌트와 런타임 introspection 동작 확인 |
| Template string literals | 기존 f-string과 역할 차이 구분 |
| Multiple interpreters | 병렬성/동시성 모델 이해 |
| Improved error messages | 오류를 읽고 원인을 추론하는 능력 강화 |
| Standard library updates | `heapq`, `pathlib`, `asyncio`, `typing` 등 item별 확인 |
| Free-threaded mode / JIT builds | CPython 구현 변화와 언어 사양의 차이 구분 |

## Compatibility Rule

문서를 작성할 때 다음 기준을 따릅니다.

1. Python 3.14.6에서의 현재 동작을 먼저 설명합니다.
2. Python 3.14 전용 문법/API는 별도 표시합니다.
3. 언어 사양(language specification)과 CPython 구현 세부사항을 구분합니다.
4. 표준 라이브러리 API는 공식 문서 링크를 남깁니다.
5. 예제는 작고 재현 가능해야 하며, 가능하면 `assert`로 의도를 드러냅니다.

## Version Badge Convention

| Badge | Meaning |
|---|---|
| `py314.6` | Python 3.14.6 기준으로 확인 |
| `language-core` | Python 언어 핵심 개념 |
| `stdlib` | 표준 라이브러리 중심 |
| `cpython-detail` | CPython 구현 세부사항 |
| `compatibility-check` | 다른 Python 버전에서 확인 필요 |
