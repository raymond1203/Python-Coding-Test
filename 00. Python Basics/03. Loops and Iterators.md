## Chapter Thesis

Python의 반복문은 단순히 index를 증가시키는 문법이 아닙니다. Python의 반복은 **iterator protocol** 위에 서 있습니다.

이 장의 핵심은 다음 질문입니다.

> “나는 지금 collection을 다루는가, iterator를 소비하는가?”

이 차이를 이해하면 `range(len(...))`를 남용하지 않고, `enumerate`, `zip`, `itertools`, generator를 자연스럽게 사용할 수 있습니다.

## Item 17. Prefer enumerate over range

### Central Question

index가 필요한 반복에서 왜 `range(len(...))`보다 `enumerate`가 자연스러울까?

### My Explanation

Python에서 반복의 기본 단위는 index가 아니라 element입니다.

```python
names = ["Ada", "Guido", "Grace"]

for name in names:
    assert isinstance(name, str)
```

index도 필요할 때는 `enumerate`가 “몇 번째인지”와 “값이 무엇인지”를 함께 제공합니다.

```python
pairs = []
for index, name in enumerate(names, start=1):
    pairs.append((index, name))

assert pairs == [(1, "Ada"), (2, "Guido"), (3, "Grace")]
```

`range(len(names))`는 index를 만든 뒤 다시 값을 조회합니다.

```python
for index in range(len(names)):
    name = names[index]
```

이 방식은 “값을 순회한다”는 Python의 기본 사고와 거리가 있습니다.

### Takeaway

값만 필요하면 직접 순회하고, index도 필요하면 `enumerate`를 사용합니다.

## Item 18. Use zip to Process Iterators in Parallel

### Central Question

여러 iterable을 같은 속도로 순회할 때, 무엇이 가장 Python다운 표현인가?

### My Explanation

`zip`은 여러 iterable에서 값을 하나씩 꺼내 tuple로 묶습니다.

```python
names = ["Ada", "Guido"]
scores = [95, 88]

result = []
for name, score in zip(names, scores):
    result.append(f"{name}:{score}")

assert result == ["Ada:95", "Guido:88"]
```

길이가 다르면 기본 `zip`은 가장 짧은 iterable에 맞춰 멈춥니다.

```python
assert list(zip([1, 2, 3], ["a"])) == [(1, "a")]
```

길이 불일치가 버그라면 `strict=True`를 고려합니다.

```python
try:
    list(zip([1, 2], ["a"], strict=True))
except ValueError:
    mismatch_detected = True
else:
    mismatch_detected = False

assert mismatch_detected
```

### Takeaway

`zip`은 병렬 순회의 의도를 드러냅니다. 길이가 같아야 한다면 `strict=True`로 가정을 코드에 남깁니다.

## Item 19. Avoid else Blocks After for and while Loops

### Central Question

loop의 `else`는 왜 많은 사람에게 낯설게 읽힐까?

### My Explanation

Python의 loop-else는 loop가 `break` 없이 끝났을 때 실행됩니다.

```python
def has_even(numbers: list[int]) -> bool:
    for number in numbers:
        if number % 2 == 0:
            break
    else:
        return False
    return True

assert has_even([1, 3, 5]) is False
assert has_even([1, 4, 5]) is True
```

문법적으로는 유효하지만, `else`라는 단어가 조건문의 `else`와 다르게 느껴져 읽는 사람이 멈칫할 수 있습니다.

명시적 조기 return이 더 읽기 쉬운 경우가 많습니다.

```python
def has_even(numbers: list[int]) -> bool:
    for number in numbers:
        if number % 2 == 0:
            return True
    return False
```

### Takeaway

loop-else는 알고 있으면 좋지만, 독자가 바로 이해하기 어렵다면 조기 return이나 helper function을 선택합니다.

## Item 20. Never Use for Loop Variables After the Loop Ends

### Central Question

`for` loop의 변수는 loop가 끝난 뒤에도 살아 있을까?

### My Explanation

Python의 `for` loop는 새 scope를 만들지 않습니다. loop variable은 loop 이후에도 남아 있습니다.

```python
for number in [1, 2, 3]:
    pass

assert number == 3
```

이 특성은 실수의 원인이 될 수 있습니다. loop 이후에 마지막 값을 써야 한다면 더 명시적으로 작성합니다.

```python
items = ["a", "b", "c"]
last_item = items[-1] if items else None

assert last_item == "c"
```

### Takeaway

loop variable은 loop의 부산물이지 결과값이 아닙니다. loop 이후 필요한 값은 별도 이름으로 명시합니다.

## Item 21. Be Defensive when Iterating over Arguments

### Central Question

함수 인자로 받은 iterable은 여러 번 순회해도 괜찮을까?

### My Explanation

list는 여러 번 순회할 수 있습니다. iterator는 한 번 소비되면 끝납니다.

```python
values = iter([1, 2, 3])

assert list(values) == [1, 2, 3]
assert list(values) == []
```

함수 인자가 iterable인지 iterator인지 모호할 때는 방어적으로 설계해야 합니다.

```python
def average(values) -> float:
    values = list(values)
    if not values:
        raise ValueError("values must not be empty")
    return sum(values) / len(values)

assert average(iter([10, 20, 30])) == 20
```

단, `list(values)`는 모든 값을 메모리에 올립니다. 큰 stream이라면 한 번의 순회로 처리해야 합니다.

```python
def streaming_sum(values) -> int:
    total = 0
    for value in values:
        total += value
    return total
```

### Takeaway

함수는 “반복 가능한 것”을 받을지 “반복자”를 받을지 의도적으로 정해야 합니다. 여러 번 순회가 필요하면 list로 고정하고, stream 처리라면 한 번만 소비합니다.

## Item 22. Never Modify Containers While Iterating over Them; Use Copies or Caches Instead

### Central Question

순회 중인 container를 바꾸면 왜 위험할까?

### My Explanation

반복 중에 list를 수정하면 index와 element의 관계가 변합니다.

```python
numbers = [1, 2, 3, 4]

for number in numbers[:]:
    if number % 2 == 0:
        numbers.remove(number)

assert numbers == [1, 3]
```

여기서는 `numbers[:]`로 복사본을 순회하고 원본을 수정했습니다. 더 좋은 방법은 새 collection을 만드는 것입니다.

```python
numbers = [1, 2, 3, 4]
odd_numbers = [number for number in numbers if number % 2 == 1]

assert odd_numbers == [1, 3]
```

`dict`에서는 key snapshot을 만들어 mutation 중에도 순회 대상이 변하지 않게 할 수 있습니다.

```python
scores = {"Ada": 10, "Guido": 0, "Grace": 20}

for name in list(scores):
    if scores[name] == 0:
        del scores[name]

assert scores == {"Ada": 10, "Grace": 20}
```

### Takeaway

순회와 mutation은 분리합니다. 복사본을 순회하거나, 새 container를 만들거나, 변경할 대상을 먼저 모읍니다.

## Item 23. Pass Iterators to any and all for Efficient Short-Circuiting Logic

### Central Question

조건 검사에서 왜 generator expression과 `any`/`all`이 잘 어울릴까?

### My Explanation

`any`는 참인 값을 만나면 멈추고, `all`은 거짓인 값을 만나면 멈춥니다.

```python
def is_even(number: int) -> bool:
    return number % 2 == 0

assert any(is_even(n) for n in [1, 3, 4, 5]) is True
assert all(is_even(n) for n in [2, 4, 6]) is True
```

list comprehension을 넘기면 전체 list를 먼저 만듭니다.

```python
result = any([is_even(n) for n in [1, 3, 4, 5]])
assert result is True
```

조건 검사는 generator expression이 더 자연스럽습니다.

```python
result = any(is_even(n) for n in [1, 3, 4, 5])
assert result is True
```

### Takeaway

`any`/`all`에는 list가 아니라 iterator를 넘겨 short-circuiting의 장점을 살립니다.

## Item 24. Consider itertools for Working with Iterators and Generators

### Central Question

반복 패턴을 직접 구현하기 전에 어떤 표준 도구를 떠올려야 할까?

### My Explanation

`itertools`는 반복 작업의 작은 building block을 제공합니다. 직접 loop를 짜기 전에 “이미 이름 붙은 반복 패턴인가?”를 생각할 수 있습니다.

```python
from itertools import accumulate, combinations, pairwise

assert list(accumulate([1, 2, 3, 4])) == [1, 3, 6, 10]
assert list(combinations(["a", "b", "c"], 2)) == [("a", "b"), ("a", "c"), ("b", "c")]
assert list(pairwise([10, 20, 30])) == [(10, 20), (20, 30)]
```

`itertools`의 강점은 lazy evaluation입니다. 필요한 만큼만 값을 꺼낼 수 있습니다.

```python
from itertools import count, islice

first_five = list(islice(count(10), 5))
assert first_five == [10, 11, 12, 13, 14]
```

### Takeaway

반복 패턴에 이름이 있다면 직접 구현보다 표준 라이브러리 추상화를 먼저 고려합니다.
