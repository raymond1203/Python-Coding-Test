# Grind 75 021. 3Sum

> 세 수의 합이 0인 조합을 찾는 문제다. 하나를 고정하고 나머지 둘을 two pointers로 찾으면 중복 제거와 탐색 효율을 동시에 잡을 수 있다.

## Link

- [LeetCode - 3Sum](https://leetcode.com/problems/3sum)

## Problem Model

배열에서 서로 다른 index의 세 값을 골라 합이 0이 되는 unique triplet을 모두 찾는다. 값 조합 기준으로 중복을 제거해야 한다.

## Pattern

- [Two Pointers](../../../03.%20Problem%20Solving%20Patterns/01.%20Two%20Pointers.md)
- [Sorting](../../../02.%20Algorithms/01.%20Sorting.md)
- [Array and List](../../../01.%20Data%20Structures/01.%20Array%20and%20List.md)

## Approach

1. 배열을 정렬한다.
2. 첫 번째 값 `nums[i]`를 고정한다.
3. 나머지 구간에서 `left`, `right`를 두고 합을 비교한다.
4. 합이 작으면 `left`를 오른쪽으로, 크면 `right`를 왼쪽으로 이동한다.
5. 합이 0이면 triplet을 기록하고 양쪽 중복 값을 건너뛴다.

## Python 3.14 Solution

```python
class Solution:
    def threeSum(self, nums: list[int]) -> list[list[int]]:
        nums.sort()
        result: list[list[int]] = []
        n = len(nums)

        for i in range(n - 2):
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            if nums[i] > 0:
                break

            left, right = i + 1, n - 1
            while left < right:
                total = nums[i] + nums[left] + nums[right]

                if total < 0:
                    left += 1
                elif total > 0:
                    right -= 1
                else:
                    result.append([nums[i], nums[left], nums[right]])
                    left += 1
                    right -= 1

                    while left < right and nums[left] == nums[left - 1]:
                        left += 1
                    while left < right and nums[right] == nums[right + 1]:
                        right -= 1

        return result
```

## Correctness Notes

- 정렬 후 `i`를 고정하면 남은 문제는 target `-nums[i]`를 만드는 two-sum이다.
- `total < 0`이면 더 큰 값이 필요하므로 `left`를 증가시키는 것이 유일하게 가능한 개선이다.
- `total > 0`이면 더 작은 값이 필요하므로 `right`를 감소시킨다.
- 고정값과 양쪽 포인터의 중복을 건너뛰므로 같은 값 triplet이 중복 기록되지 않는다.

## Complexity

- Time: `O(n^2)`
- Space: `O(1)` auxiliary space, excluding output.

## Mistake Notes

- 정렬하지 않으면 중복 제거가 훨씬 복잡해진다.
- 정답 triplet을 찾은 뒤에는 `left`, `right` 양쪽 중복을 모두 건너뛰어야 한다.
- `nums[i] > 0`이면 이후 값도 모두 양수라 합이 0이 될 수 없으므로 종료 가능하다.
